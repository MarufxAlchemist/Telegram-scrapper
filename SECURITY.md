## Reporting a Vulnerability

We take the security of our project seriously. If you find any security vulnerability, please report it to us as soon as possible.

### How to Report

Please **DO NOT** create a public GitHub issue for security vulnerabilities. Instead, please report them via email to contact@example.com.

We will review your report and respond within 48 hours. If the vulnerability is accepted, we will prioritize a fix and release a patch as soon as possible.

### What to Include

When reporting a vulnerability, please include the following information:

* A description of the vulnerability.
* Steps to reproduce the vulnerability.
* Any relevant logs or error messages.
* Your contact information (optional, but requested for credit).

## Disclosure Policy

We follow a responsible disclosure policy. This means that we ask you to give us a reasonable amount of time to fix the vulnerability before making it public. We will do our best to fix the vulnerability as soon as possible, and we will keep you updated on our progress.

## Acknowledgements

If you report a vulnerability that leads to a fix, we will acknowledge your contribution in our security advisories and release notes (unless you wish to remain anonymous).
